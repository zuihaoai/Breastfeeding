# 母乳喂养电子书导入方案

> 这份方案是把 `docs/母乳喂养/` 导入到 `zuihaoai/Breastfeeding` 的执行蓝图。它和 [00-目录总表.md](./00-目录总表.md) 互相配套：目录总表负责“内容怎么分”，这份方案负责“怎么落成电子书”。

## 目标

- 把母乳喂养知识库做成可阅读、可检索、可复用的电子书仓库。
- 让后续新增内容只需要增量追加，不需要推翻旧结构。
- 为成长AI和奶爸AI预留统一的章节编号、资源位和检索锚点。

## README 组织规则

- `README.md` 顶部放电子书名称、阅读入口、版本信息。
- 紧接着放一段“这本书适合谁看”的说明。
- 再放一份目录，目录必须和 `00-目录总表.md` 的章号一致。
- 每个章节用 `##` 作为章标题，内部继续用 `###` 和 `####` 细分。
- 如果后续增加新章，只能在末尾追加新的章号，避免破坏已发布版本的锚点。

## 自我进化规则

- 新知识优先补到对应章节，不轻易创建平行重复章节。
- 若某个主题反复出现，先在目录总表里建立固定索引，再拆正文。
- 每次更新后保留变更记录，至少记录“新增了什么”“替换了什么”“为什么改”。
- 电子书正文、目录总表、AI 检索标签三者必须保持同号同名。

## 资源预留

- 图片统一放在 `assets/images/`，文件名建议使用 `01-...png` 这种章号前缀。
- 视频统一放在 `assets/videos/`，同样使用章号前缀。
- 资源引用建议优先使用“章号 + 节点名”组合，方便 AI 直接定位。

## 下一步执行顺序

1. 根据 [00-目录总表.md](./00-目录总表.md) 生成电子书 `README.md` 的正文框架。
2. 把每一章的内容拆成可直接粘入 `README.md` 的 Markdown 段落。
3. 再补封面、索引页和导出脚本。
4. 最后再进入插图与视频的映射阶段。
