# 母乳喂养发布同步清单

> 这份清单用于把 `tmp/breastfeeding-books/` 的 staging 成果同步到 `zuihaoai/Breastfeeding`。当前本地已经完成目录、正文骨架、媒体映射、构建脚本和 release 打包。

## 当前可交付物

- 目录总表：`00-目录总表.md`
- 电子书导入方案：`01-电子书导入方案.md`
- 插图视频映射方案：`02-插图视频映射方案.md`
- 媒体资源待办：`tmp/breastfeeding-books/media.backlog.md`
- 整书生成入口：`tmp/breastfeeding-books/book.md`
- 发布包：`tmp/breastfeeding-books/dist/Breastfeeding-release.zip`

## 同步目标

- 把 staging 仓库中的 `README.md`
- 把 `book.md`
- 把 `chapters/`
- 把 `docs/`
- 把 `meta/`
- 把 `assets/`
- 把 `tools/`

统一同步到 `zuihaoai/Breastfeeding`。

## 同步前检查

- 确认远端仓库可访问。
- 确认目标仓库根目录没有冲突文件。
- 确认 `manifest.json` 与 `book.md` 的章号一致。
- 确认 `media.backlog.md` 已生成。

## 同步后检查

- `python3 tools/build_book.py` 可以重新生成 `book.generated.md`
- `python3 tools/build_media_backlog.py` 可以重新生成 `media.backlog.md`
- `python3 tools/package_release.py` 可以重新生成发布包
- 目录总表与正文标题一致

## 后续完成标准

- 发布包能直接被 GitHub 仓库接收。
- 电子书目录、正文、媒体资源三层编号一致。
- 后续新增内容只需要增量更新章节或媒体清单。
